# Prerequisites to Use the Rawi CLI

To use the Rawi CLI, you need to meet a few prerequisites depending on the AI provider you want to access. Rawi supports multiple providers, each with its own requirements for credentials and setup.

## System Requirements

- **Node.js**: Version 18.0.0 or higher
- **Package Manager**: npm, pnpm, or yarn
- **Operating System**: Linux, macOS, or Windows

## Provider Credentials

To access AI services with Rawi, you need credentials for the provider you want to use. For security, do not share your API keys or credentials.

| Provider       | Prerequisite Account / Credentials          | Notes                                                                                 |
| -------------- | ------------------------------------------- | ------------------------------------------------------------------------------------- |
| OpenAI         | OpenAI account, API key                     | Get API key from [OpenAI Platform](https://platform.openai.com/api-keys)              |
| Anthropic      | Anthropic account, API key                  | Get API key from [Anthropic Console](https://console.anthropic.com/)                  |
| Google Gemini  | Google account, API key                     | Get API key from [Google AI Studio](https://aistudio.google.com/)                     |
| Ollama (local) | None (runs locally)                         | Install [Ollama](https://ollama.ai/) and pull models                                  |
| Azure OpenAI   | Azure subscription, API key, resource name  | See [Azure OpenAI docs](https://learn.microsoft.com/azure/cognitive-services/openai/) |
| Amazon Bedrock | AWS account, IAM credentials or access keys | Use IAM user or AWS credential chain                                                  |
| Qwen (Alibaba) | Alibaba Cloud account, API key              |                                                                                       |
| xAI            | xAI account, API key                        |                                                                                       |

## Security Best Practices

- **Do not use root or admin credentials for daily tasks.**
- **Create provider-specific users or API keys with least privilege.**
- **Store credentials securely (Rawi stores them encrypted in `~/.rawi/`).**
- **For AWS Bedrock, use IAM roles or the AWS credential provider chain.**

## Next Steps

After you have the required credentials:

<Steps>

1. **Install Rawi CLI:**

   <Tabs>
      <TabItem label="npm" icon="npm">

         ```bash
         npm install -g rawi
         ```

      </TabItem>
      <TabItem label="pnpm" icon="pnpm">

         ```bash
         pnpm add -g rawi
         ```

      </TabItem>
      <TabItem label="yarn" icon="seti:yarn">

         ```bash
         yarn global add rawi
         ```

      </TabItem>
      <TabItem label="bun" icon="bun">

         ```bash
         bun add -g rawi
         ```

      </TabItem>
   </Tabs>

2. **Configure your first provider (interactive):**

   ```bash
   rawi configure
   ```

   Or configure via command line for automation:

   ```bash
   rawi configure --provider openai --api-key <your-key> --model gpt-4o
   ```

3. **Verify installation:**

   ```bash
   rawi --version
   rawi info
   ```

4. **Start using Rawi:**
   ```bash
   rawi ask "What is TypeScript?"
   ```

</Steps>
